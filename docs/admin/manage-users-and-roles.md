# Manage users and roles
Use this article to invite users, change user roles, and remove users from SensorHub Cloud.

Users are people who can sign in to your SensorHub Cloud account. Roles control what each user can view or change.

## Before you begin
Make sure you have:

- A SensorHub Cloud account.
- Permission to manage users.
- The email address for each user you want to invite.
- The role each user should have.

Only account owners and administrators can manage users.

## Choose the right role
Give each user the lowest role they need to do their work.

This helps protect your account.

For example:

- A manager who owns the account may need the **Owner** role.
- A technician who adds devices may need the **Administrator** role.
- A team member who responds to alerts may need the **Operator** role.
- A user who only checks reports may need the **Viewer** role.

## Invite a user
To invite a user:

1. Sign in to SensorHub Cloud.
2. From the left menu, select **Admin.**
3. Select **Users.**
4. Select **Invite user.**
5. Enter the user's email address.
6. Choose a role.
7. Optional: Choose one or more locations the user can access.
8. Select **Send invite.**

SensorHub Cloud sends an email invite to the user.

The user must accept the invite before they can sign in.

### Resend an invite
If a user does not receive the invite, you can send it again.

To resend an invite:

1. From the left menu, select **Admin.**
2. Select **Users.**
3. Find the invited user.
4. Select the **More actions** menu.
5. Select **Resend invite.**

Ask the user to check their inbox and spam folder.

## Change a user role
You can change a user's role when their job changes.

To change a user role:

1. From the left menu, select **Admin.**
2. Select **Users.**
3. Find the user.
4. Select the user's name.
5. Select **Edit** role.
6. Choose the new role.
7. Select **Save.**

The new role takes effect the next time the user refreshes the page or signs in.

## Change location access
Some users only need access to certain locations.

For example, a technician may only need access to the North Building location.

To change location access:

1. From the left menu, select **Admin.**
2. Select **Users.**
3. Find the user.
4. Select the user's name.
5. Select **Location access.**
6. Choose the locations the user can access.
7. Select **Save.**

If a user does not have access to a location, they cannot view its devices, readings, or alerts.

## Disable a user
Disable a user when you want to block sign-in but keep the user record.

This can be useful when a user is on leave or when access needs to be paused.

To disable a user:

1. From the left menu, select **Admin.**
2. Select **Users.**
3. Find the user.
4. Select the **More actions** menu.
5. Select **Disable user.**
6. Select **Disable** to confirm.

The user can no longer sign in.

### Enable a disabled user
To enable a disabled user:

1. From the left menu, select **Admin.**
2. Select **Users.**
3. Open the **Status** filter.
4. Select **Disabled.**
5. Find the user.
6. Select the **More actions** menu.
7. Select **Enable user.**

The user can sign in again.

## Remove a user
Remove a user when they should no longer have access to SensorHub Cloud.

To remove a user:

1. From the left menu, select **Admin.**
2. Select **Users.**
3. Find the user.
4. Select the **More actions** menu.
5. Select **Remove user.**
6. Review the warning message.
7. Select **Remove** to confirm.

Removing a user does not delete sensor readings, alerts, or device history.
