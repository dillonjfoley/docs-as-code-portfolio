# Confidentiality Note
This repository is a fictional documentation project created for portfolio purposes.

The product, company, workflows, endpoints, screenshots, examples, release notes, and support scenarios in this repository are fictional. They were created to demonstrate technical writing, information architecture, docs-as-code organization, API documentation, troubleshooting content, release documentation, and documentation governance.

## No proprietary content
This repository does not include:
- Proprietary documentation from any current or former employer.
- Internal company processes.
- Private product information
- Customer data.
- Internal screenshots.
- Real API keys, tokens, or credentials.
- Non-public API endpoints.
- Confidential architecture diagrams.
- Employer-owned source files.
- Unreleased product details.

Any resemblance to a real product, company, workflow, or system is unintentional.

## Purpose of this project
This project is designed to show how I approach documentation work, including:

- Structuring a documentation set.
- Writing task-based user documentation.
- Creating API documentation.
- Organizing troubleshooting content.
- Writing release notes.
- Maintaining consistent terminology.
- Using Markdown and GitHub in a docs-as-code workflow.

The goal is to demonstrate documentation judgment and workflow, not to reproduce work from a previous employer.
